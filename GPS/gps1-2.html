What is pair programming?

  • Collaborative work on writing code
  • Usually done between two programmers working on one project
  • Can be done physically or remotely

Why is pair programming important to DBC?

  • Can learn from your pair
  • Real world practice , good to be ready for it
  • Allows for feedback after pairing session that helps you improve

Driver vs Navigator

  • Navigator is the like a co-pilot, looking ahead, helping to test code and research code
  • Driver is in charge of implementation, but not only typing, can put in what he thinks works

Our experience

  • So far only a few guided pairing sessions - its been fun
  • Team feeling
