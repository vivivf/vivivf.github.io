Margin, Border & Padding What are the differences?

October 19, 2014

During this week, we were introduced to HTML and CSS and our challenge was to create a personal website. The website you are navigating now is the result of this challenge :) If you have never created a website from the scratch and have a ton of doubts about how to do some formatting, you are in the right place! In this blog post I'll cover only some of the CSS properties, but it is a good start for those who want to understand some basic concepts! As most of you probably know, HTML stands for Hyper Text Markup Language and it is used to make content available in a webpage. The CSS (Cascading Style Sheets) is responsible for describing the look and formatting of a document written in a markup language. These are veeeery high level definitions, but it is not the objective of this blog post to talk extensively about HTML and CSS, since this is a subject for a whole book, so let's focus on 3 important CSS properties: margin, padding and border. This was the first time I used CSS, and to be honest, in the beginning I was a little bit lost. I kept asking myself "What the hell I'm doing wrong? Why can't I move or position my elements? Why do I mess everything when I try to configure only one element". If you have the same doubts, I hope this blog will help you to clarify them. The first important thing to understand when talking about visual formatting is the box model. According to the box model, every element in the page is a rectangular box. Each box is composed by the content itself (image, text, etc), surrounded by the padding, border and margin. The figure bellow illustrates the box model. As you can see, the margin, border and padding can be broken down into top, right, bottom and left, which means that you can configure each one individually. Now that the box model was presented, let's move to the properties. Padding It is the space between the content and the border. Let's say you are going to create a menu and you want to have a text with a border to represent your menu options, but you don't want that border to be too close to the text. In this case you can use the padding to add space between the text and the border. Take a look in the example bellow: This is the menu of this website. As you can see, the space between the border and the top/bottom is different from the space between the border and the left/right edges. This is because I configured the padding of 0.1em for the bottom and top and 0.8em for the left and right. The padding property accepts as input from 1 to 4 values. If only one value is specified it will apply the same padding for all the sides, so "padding:3em" means that 3em will be added between the border and all the sides of the content. If more than one value is specified, it starts at the top and then moves clockwise until it reaches the left. If 2 values are given, like in the image, it will apply the first value for the top and bottom and the second value for the right and left. The padding value can also be a percentage that is calculated with respect to the width of the generated box's containing block. Some more examples: As mentioned before, you can configure the padding for each side individually, so if you donâ€™t want to configure all the sides, you can use the following properties: padding-top padding-right padding-bottom padding-left Border: The border is right in the middle of the box model and it indicates the "edge" between padding and margin. It can also be broken down into border-top, border-right, border-bottom and border-left. The border characteristics are: width, color, and style and they can be configured for every element. The available values for the width are: thin, medium, thick and , when the border's thickness has an explicit value. Explicit border widths cannot be negative. For the color there are only two options, one is to specify a color either by name or by its hexadecimal code and the other one is to use "transparent". In this case the border will be transparent, but it will exist if the width is different from zero. The style characteristic has the following options: none: No border; the computed border width is zero. hidden: Same as 'none', except in terms of border conflict resolution for table elements. dotted: The border is a series of dots. dashed: The border is a series of short line segments. solid: The border is a single line segment. double: The border is two solid lines. The sum of the two lines and the space between them equals the value of 'border-width'. groove: The border looks as though it were carved into the canvas. ridge: The opposite of 'groove': the border looks as though it were coming out of the canvas. inset: The border makes the box look as though it were embedded in the canvas. outset: The opposite of 'inset': the border makes the box look as though it were coming out of the canvas. Since a picture is worth a thousand words, see the example bellow: Another interesting property is the border-radius, which enables us to round the corners of an element. It accepts length units and percentage. A single value will round all four corners of an element equally; two values will round the top-left/bottom-right and top-right/bottom-left corners in that order; four values will round the top-left, top-right, bottom-right, and bottom-left corners in that order. It can also be broken down for each side of the box, but in this case the options are: top-left, top-right, bottom-right, bottom-left. Here are some examples: div { border-radius: 5px; } div { border-radius: 50%; } div { border-radius: 15px 75px; } Margin: And we are finally in the last piece of the box model, the margin. It is very similar to the padding but instead of being the space between the element and the border, it is the space between the border and the other elements. The margin property allows us to set the amount of space that surrounds an element. As padding and border, the margin also can be broken down into top, right, bottom and left. The margin value can be specified as the following: Specifies a fixed width. Negative values are allowed. The percentage is calculated with respect to the width of the generated box's containing block. Note that this is true for 'margin-top' and 'margin-bottom' as well. auto: Here's a visual demonstration of CSS margins: As the padding, the margin can take from 1 to 4 values, and it uses the same order as padding, starting from the top and moving clockwise. Here are some code examples to solidify the understanding: body { margin: 2em } /* all margins set to 2em */ body { margin: 1em 2em } /* top & bottom = 1em, right & left = 2em */ body { margin: 1em 2em 3em } /* top=1em, right=2em, bottom=3em, left=2em */ body { /*each margin configured individually*/ margin-top: 1em; margin-right: 2em; margin-bottom: 3em; margin-left: 2em; } Well, this is it for today!
