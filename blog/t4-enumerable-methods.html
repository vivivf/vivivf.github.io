Map Enumerable Method

November 2, 2014

Last week we talked about collection objects, more specifically about arrays and hashes. This week we'll talk about a method called map. This method is part of the Enumerable module, which is widely used by collection objects. But what is a module? In very few words, a module is a way to pack a bunch of methods and constants together. Ruby's classes can include modules, which means that the objects of a class that includes a module can call the module's methods. Array and hash classes include the Enumerable module, therefore, any array or hash object can call any enumerable method. Having that said, we know that arrays and hashes can call the method map. The map method is very useful. It iterates through all the elements of the collection object and creates a different collection object based on the result of the operation specified in the block. Here is the syntax of the command: map {|item| block} Take a look at the example bellow to clarify how the method works: 2.0.0-p451 :629 > [1,2,3].map{|item| item.to_s} => ["1", "2", "3"] In this case, the map method will iterate through each array element and execute the "item.to_s" operation for each one. The result will be a new array with all the numbers transformed in strings. The same result can be achieved with the shorthand notation: 2.0.0-p451 :632 > [1,2,3].map(&:to_s) => ["1", "2", "3"] As you can see, the result is the same. For ruby beginners maybe the first option is easier, since it is more readable. As Im still not very used with shorthands, I personally prefer the first option, but is always good to know that there are other ways to do the same thing. Let's see one more example. 2.0.0-p451 :006 > names=["erick","jonas","kate","julie"] => ["erick", "jonas", "kate", "julie"] 2.0.0-p451 :007 > names.map{|name| name.capitalize} => ["Erick", "Jonas", "Kate", "Julie"] 2.0.0-p451 :008 > names => ["Erick", "Jonas", "Kate", "Julie”] In this example, we used the capitalize method inside the block to capitalize the names. Note that the original array was not modified, which means that map is a non-destructive method. If you want to modify the array, the method map! should be used. Check it out: 2.0.0-p451 :004 > names.map!{|name| name.capitalize} => ["Erick", "Jonas", "Kate", "Julie"] 2.0.0-p451 :005 > names => ["Erick", "Jonas", "Kate", "Julie”] Those are simple examples, but the map method can be very powerful if used wisely. This is it for today. I hope you enjoyed the reading!
