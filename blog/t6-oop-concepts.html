Class x Module

November 9, 2014

We have talked about classes last week but only to refresh our memory here is one of the definitions for class: "In object-oriented programming, a class is a construct that is used to create instances of itself - referred to as class instances, class objects, instance objects or simply objects. A class defines constituent members which enable its instances to have state and behavior." Modules are similar to classes, which means they also enable objects to have behavior through the definition of methods, the difference is that unlike classes, modules don't have instances. An object and/or a class that includes a module, will have access to its methods and constants. Modules can provide functionality across multiple classes and/or objects so, they are perfect to be used when objects from different classes have to implement the same behavior. A good example is the Math module. It has constants, like Pi, and methods that define mathematical calculations, like sqrt(number), which returns the square foot of the specified number. The Math module can be used by class/object that includes it. Let's see how a method is defined and how to include it. module FirstModule def testing_module p "Testing our first module" end end This is how a method definition looks like. It is very similar to the class, we only replaced the word "class" by "module" So now, we are going to include this method in a class: class ModuleTester include FirstModule end And finally, we'll create an instance of the class ModuleTester and try to call the method testing_module module_test = ModuleTester.new module_test.testing_module Here is the output: 2.0.0-p451 :543 > "Testing our first module" As you can see, the module_test object can access the method testing_module. This is, of course, a very simple demonstration of the use of modules, but it serves to show that modules can expand considerably the set of messages to which an object can respond. As Ruby provides single inheritance, where a subclass is allowed only one parent superclass, modules play an important role when it comes to allowing the objects to have a broader scope.
